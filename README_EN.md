## Target Hardware

| Item | Model |
|------|-------|
| Drone | DJI Mini 4 Pro |
| Remote Controller | DJI RC-N2 |

## Features

### Mission Modes

- **Mission A (Hover Recording + Photos)**: Fly to target coordinates -> 360-degree yaw rotation recording -> 7 multi-angle photos -> Return home
- **Mission B (Orbit Recording)**: Fly to target coordinates -> Orbit around target at specified radius with 360-degree recording -> Return home

### Core Capabilities

- MQTT remote command control (takeoff, return home, cancel, reset, mission start)
- Real-time status reporting (coordinates, altitude, battery, GPS, mission state)
- Gimbal angle control (independent settings for recording/photos)
- Safety protection (GPS signal check, battery monitoring, flight boundary limits)
- Automatic photo download after mission completion

## Quick Start

### Requirements

- Android Studio (2024.1+)
- JDK 17+
- Android SDK 35
- DJI Developer Account ([developer.dji.com](https://developer.dji.com))

### Setup Steps

1. **Clone the repository**
   ```bash
   git clone <repo-url>
   ```

2. **Apply for DJI API Key**
   - Go to [DJI Developer Platform](https://developer.dji.com) and register
   - Create an application to obtain an API Key
   - Package Name: `com.example.djimission`

3. **Configure gradle.properties**
   ```bash
   cp gradle.properties.example gradle.properties
   ```
   Edit `gradle.properties` and fill in your API Key:
   ```properties
   AIRCRAFT_API_KEY = your_api_key_here
   ```

4. **Open the project**
   - Open the project root directory in Android Studio
   - Wait for Gradle sync to complete

5. **Build and Run**
   ```bash
   ./gradlew assembleDebug
   ```
   Or click Run directly in Android Studio.

## MQTT Communication

| Topic | Direction | Purpose |
|-------|-----------|---------|
| `{prefix}/command` | Backend -> App | Flight commands (takeoff/rtl/cancel/reset) |
| `{prefix}/mission/start` | Backend -> App | Mission start (A/B mode) |
| `{prefix}/status` | App -> Backend | Flight status reporting |
| `{prefix}/ack` | App -> Backend | Command execution acknowledgement |
| `{prefix}/heartbeat` | App -> Backend | Connection heartbeat |

### Mission Start Payload

```json
{
  "msg_id": "msn_001",
  "mission": "A",
  "target": { "lat": 0.0, "lng": 0.0, "alt": 10.0 },
  "orbit": { "radius": 0, "alt": 5.0, "gimbal_pitch": -45, "photo_gimbal_pitch": -30 },
  "photo_count": 7,
  "timestamp": 1700000002000
}
```

- `mission`: `"A"` (hover recording + photos) or `"B"` (orbit recording, requires `radius > 0`)
- `orbit.radius`: Orbit radius in meters, 0 = yaw rotation in place
- `orbit.alt`: Orbit altitude in meters, null = use target.alt
- `orbit.gimbal_pitch`: Camera gimbal pitch during recording (degrees)
- `orbit.photo_gimbal_pitch`: Camera gimbal pitch during photos (degrees), Mission A only
